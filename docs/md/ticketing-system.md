# Система продажи билетов — варианты и анализ

## Обзор вариантов

### 1. Платформа-посредник

| Платформа | Плюсы | Минусы | Комиссия |
|---|---|---|---|
| **Eventbrite** | Известный бренд, виджет встраивается на сайт | ~€22 с билета за €590 | ~3.7% + €0.79 |
| **Luma** | Красивый, бесплатный (берёт с покупателя) | Меньше кастомизации, менее известен в Европе | С покупателя |
| **Pretix** | Open-source, немецкий (GDPR из коробки), self-hosted бесплатно | Нужен сервер, настройка, поддержка | 0% (self-hosted) |
| **Tito** | Чистый UI, заточен под конференции | Комиссия + Stripe fees | ~3% + Stripe |

При 100 билетах по €500 среднему — €1500–3000 в комиссиях.

### 2. Stripe Payment Links (без кода)

- Создаёшь продукт в Stripe Dashboard → получаешь ссылку → вставляешь как кнопку на сайт.
- Ноль кода, работает сразу.
- Можно добавить промокоды, early bird цены.
- **Минус**: нет управления участниками, нет кастомных писем с билетами.

### 3. Stripe + простой бэкенд (рекомендуемый)

Минимальная комиссия, полный контроль, автоматизация.

## Stripe + бэкенд — детальный разбор

### Схема работы

```
Пользователь нажимает "Buy" на сайте
        ↓
Stripe Checkout (hosted страница Stripe)
        ↓
Пользователь платит (карта, Apple Pay, Google Pay)
        ↓
Stripe шлёт webhook на бэкенд
        ↓
Бэкенд: записывает покупателя + отправляет email
        ↓
Пользователь возвращается на сайт → страница "Спасибо"
```

### Что нужно написать

**Фронтенд (на сайте):**
- Кнопка «Buy Ticket» → редирект на Stripe Checkout Session
- Страница «Спасибо» после оплаты
- ~20–30 строк JS

**Бэкенд (serverless function) — два эндпоинта:**

1. `POST /create-checkout-session` — создаёт Stripe Checkout Session с нужным тарифом, перенаправляет пользователя на оплату
2. `POST /webhook` — принимает событие `checkout.session.completed` от Stripe, сохраняет данные покупателя, отправляет email

### Где хостить бэкенд

| Вариант | Бесплатный план | Сложность | Примечание |
|---|---|---|---|
| **Cloudflare Workers** | 100k запросов/день | Низкая | Быстрый, edge, KV для хранения |
| **Vercel Functions** | 100k запросов/мес | Низкая | Хорошо если переедете на Next.js |
| **Netlify Functions** | 125k запросов/мес | Низкая | Аналог Vercel |
| **Google Apps Script** | Бесплатно | Самая низкая | Можно писать прямо в Google Sheets |
| **VPS на IONOS** | Уже есть хостинг | Средняя | Нужен Node.js/Python на сервере |

### Хранение данных покупателей

| Вариант | Подходит для | Примечание |
|---|---|---|
| **Google Sheets** (через API) | 100–300 билетов | Самый простой, удобно смотреть |
| **Cloudflare KV / D1** | Если бэкенд на CF Workers | Всё в одной экосистеме |
| **Airtable** | Красивый UI | Бесплатно до 1000 записей |
| **Stripe Dashboard** | Минимум усилий | Stripe сам хранит все транзакции |

### Отправка email с подтверждением

- **Stripe receipt** — включается в Dashboard одной галочкой (чек об оплате)
- **Кастомное письмо с билетом**: Resend (бесплатно 100 писем/день), SendGrid, или Formspree (уже подключён на сайте)

## Стоимость в проде

| Статья | Цена |
|---|---|
| Stripe (EU карта) | 1.5% + €0.25 за транзакцию |
| Stripe (non-EU карта) | 2.5% + €0.25 за транзакцию |
| Хостинг бэкенда | €0 (free tier) |
| Email | €0 (Stripe receipts или Resend free tier) |

Для сравнения: Eventbrite взял бы ~€22 с того же билета (~4.4%).

## Риски и подводные камни

1. **PSD2 / SCA** — в Европе обязательна двухфакторная аутентификация платежей. Stripe решает это из коробки.
2. **Возвраты (refunds)** — нужна чёткая политика. Stripe позволяет делать возвраты, но комиссию Stripe не возвращает.
3. **Invoicing / Rechnungen** — B2B клиенты (компании) захотят счёт с VAT. Нужно продумать, кто и как выставляет.
4. **GDPR** — Stripe хранит платёжные данные у себя, сайт не касается карт. Но данные покупателей (имя, email) в своей базе — ответственность на нас.
5. **VAT** — продажа билетов в Германии: 19% VAT. Stripe не берёт на себя VAT handling — нужно считать и отчитываться самим.
